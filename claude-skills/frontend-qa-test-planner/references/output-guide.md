# CSV 출력 작성 가이드

이 문서는 QA 테스트 케이스 CSV를 작성할 때의 상세 원칙과 예시를 제공합니다.

## CSV 기본 구조

모든 출력은 아래 4개 컬럼을 포함하는 CSV 형식:

```csv
분류,점검 항목,테스트 데이터,기대 결과
```

## 컬럼별 작성 원칙

### 1. 분류 (Category)

**형식:** `기능/이벤트명 (이벤트타입)`

**이벤트 타입 목록:**
- `ONLOADED`: 페이지/컴포넌트 로드 시
- `ONCLICK`: 클릭 이벤트
- `ONCHANGE`: 값 변경 이벤트 (셀렉트박스, 체크박스 등)
- `ONINPUT`: 텍스트 입력 이벤트
- `ONKEYPRESS`: 키보드 입력 이벤트
- `ONSUBMIT`: 폼 제출 이벤트
- `VALIDATION`: 유효성 검사
- `ONERROR`: 에러 처리
- `ONLOADING`: 로딩 상태 처리
- `EDGE`: 엣지 케이스

**예시:**
```csv
초기화면 (ONLOADED)
검색버튼 (ONCLICK)
날짜필터 (ONCHANGE)
검색어입력 (ONINPUT)
엔터키 (ONKEYPRESS)
유효성검사 (VALIDATION)
엣지케이스 (EDGE)
```

### 2. 점검 항목 (Test Item)

**형식:** "~~ 확인" 형태

**작성 원칙:**
- 검증하려는 목적을 명확히 표현
- 구체적이고 측정 가능한 항목
- 모호한 표현 지양

**Good Examples:**
```
✅ "초기 검색 조건 기본값 설정 확인"
✅ "필수 입력값 미입력 시 유효성 검사 확인"
✅ "페이지 번호 클릭 시 목록 갱신 확인"
✅ "API 호출 실패 시 에러 메시지 표시 확인"
```

**Bad Examples:**
```
❌ "초기화 테스트" (너무 모호)
❌ "버튼 동작" (무엇을 검증하는지 불명확)
❌ "데이터 확인" (구체성 부족)
```

### 3. 테스트 데이터 (Test Data)

**형식:** 테스터가 **즉시 실행 가능한** 구체적인 값

**작성 원칙:**
- 실제 입력 가능한 값 제시
- 날짜는 상대적 표현 사용 (오늘, 7일 전, 1개월 후 등)
- 선택/클릭 대상은 명확히 지정
- 빈 값, null, undefined도 명시

**Good Examples:**
```
✅ "시작일: 7일 전, 종료일: 오늘"
✅ "빈 값"
✅ "특수문자 포함 문자열: !@#$%"
✅ "2페이지 클릭"
✅ "검색어: '테스트', 필터: '전체'"
✅ "페이지 사이즈: 50건 선택"
```

**Bad Examples:**
```
❌ "적절한 날짜" (구체성 부족)
❌ "이상한 값" (모호함)
❌ "페이지 이동" (어느 페이지인지 불명확)
```

### 4. 기대 결과 (Expected Result)

**형식:** 관찰 가능한 UI/상태 변화를 구체적으로 기술

**작성 원칙:**
- **관찰 가능한** 변화만 기술
- UI 요소, 메시지, 데이터 변화 명시
- "~~된다", "~~표시된다", "~~변경된다" 형태
- 내부 로직이 아닌 사용자가 볼 수 있는 결과

**Good Examples:**
```
✅ "오류 메시지 '필수 항목입니다' 표시"
✅ "페이지가 1페이지로 초기화되고 그리드 데이터가 갱신된다"
✅ "로딩 스피너가 표시되고 완료 후 사라진다"
✅ "버튼이 비활성화 상태로 변경된다"
✅ "테이블이 날짜 기준 내림차순으로 정렬된다"
```

**Bad Examples:**
```
❌ "API가 호출된다" (내부 동작, 관찰 불가)
❌ "state가 업데이트된다" (개발자 관점, QA 관점 아님)
❌ "정상 동작한다" (너무 모호)
❌ "에러가 발생한다" (어떤 에러인지 불명확)
```

## 다중 파일 분석 시 형식

여러 파일을 분석하는 경우 **페이지별 구분**:

```csv
## 페이지 1: /frontend/pages/dashboard/index.vue
분류,점검 항목,테스트 데이터,기대 결과
초기화면 (ONLOADED),대시보드 위젯 로드 확인,페이지 진입,4개의 통계 위젯이 표시된다.
버튼클릭 (ONCLICK),새로고침 버튼 클릭 시 데이터 재조회 확인,새로고침 버튼 클릭,모든 위젯 데이터가 최신 값으로 갱신된다.

## 페이지 2: /frontend/pages/user/profile.vue
분류,점검 항목,테스트 데이터,기대 결과
초기화면 (ONLOADED),사용자 프로필 정보 로드 확인,페이지 진입,로그인한 사용자의 이름과 이메일이 표시된다.
폼입력 (ONINPUT),이름 변경 시 입력 반영 확인,이름 필드에 '홍길동' 입력,입력한 이름이 실시간으로 표시된다.
```

## 테스트 상세도별 작성 가이드

### Simple (기본)

**범위:** 핵심 기능과 주요 사용자 플로우만

**포함 항목:**
- 페이지 초기화 (ONLOADED)
- 주요 버튼 클릭 (조회, 저장, 삭제 등)
- 기본 유효성 검사 (필수값)
- 주요 데이터 조회/표시

**예시:**
```csv
분류,점검 항목,테스트 데이터,기대 결과
초기화면 (ONLOADED),초기 검색 조건 기본값 설정 확인,페이지 진입,"시작일은 7일 전, 종료일은 오늘로 설정된다."
버튼클릭 (ONCLICK),조회 버튼 클릭 시 데이터 조회 확인,조회 버튼 클릭,검색 조건에 맞는 데이터가 그리드에 표시된다.
유효성검사 (VALIDATION),필수 입력값 미입력 시 검증 확인,빈 값으로 조회 시도,"오류 메시지 '필수 항목입니다' 표시"
```

### Detailed

**범위:** Simple + 예외 케이스 + 상태 관리

**추가 항목:**
- 엣지 케이스 (빈 결과, 최대값, 최소값)
- 에러 처리 (API 실패, 네트워크 오류)
- 페이지네이션, 정렬
- 폼 유효성 검사 (형식, 범위)

**예시:**
```csv
분류,점검 항목,테스트 데이터,기대 결과
에러처리 (ONERROR),API 호출 실패 시 처리 확인,네트워크 오류 발생,사용자에게 오류 메시지가 표시되고 로딩 상태가 해제된다.
엣지케이스 (EDGE),조회 결과 0건인 경우 확인,존재하지 않는 조건으로 검색,"""조회된 데이터가 없습니다"" 메시지가 그리드에 표시된다."
페이지네이션 (ONCHANGE),페이지 번호 클릭 시 목록 갱신 확인,2페이지 클릭,해당 페이지의 데이터로 그리드가 갱신된다.
정렬 (ONCLICK),컬럼 헤더 클릭 시 정렬 동작 확인,날짜 컬럼 헤더 클릭,날짜 기준 오름차순/내림차순 정렬이 적용된다.
```

### Max

**범위:** Detailed + UI/UX 세부사항 + 접근성 + 모든 상호작용

**추가 항목:**
- 로딩 상태 UI (스피너, 스켈레톤)
- 버튼 활성화/비활성화 상태
- 툴팁, 도움말 표시
- 키보드 네비게이션
- 반응형 레이아웃
- 애니메이션/트랜지션
- 포커스 관리
- 브라우저 뒷버튼 동작

**예시:**
```csv
분류,점검 항목,테스트 데이터,기대 결과
로딩상태 (ONLOADING),데이터 로딩 중 UI 처리 확인,API 응답 대기 중,로딩 스피너 또는 스켈레톤 UI가 표시된다.
버튼상태 (ONCLICK),조회 중 버튼 비활성화 확인,조회 버튼 클릭 후 응답 전,조회 버튼이 비활성화되고 '조회 중...' 텍스트가 표시된다.
키보드네비게이션 (ONKEYPRESS),Tab 키로 포커스 이동 확인,Tab 키 반복 입력,입력 필드와 버튼 순서대로 포커스가 이동한다.
반응형 (EDGE),모바일 화면에서 레이아웃 확인,화면 너비 375px,테이블이 세로 스크롤 가능한 형태로 변경된다.
```

## 실전 예시: 검색 페이지

전체적인 흐름을 보여주는 종합 예시:

```csv
분류,점검 항목,테스트 데이터,기대 결과
초기화면 (ONLOADED),초기 검색 조건 기본값 설정 확인,페이지 진입,"시작일은 10년 전, 종료일은 오늘로 설정된다."
초기화면 (ONLOADED),초기 데이터 조회 수행 확인,페이지 진입,기본 검색 조건으로 그리드 데이터가 로드된다.
버튼클릭 (ONCLICK),검색 버튼 클릭 시 조회 동작 확인,검색 버튼 클릭,입력된 조건으로 데이터가 조회되고 그리드가 갱신된다.
엔터키 (ONKEYPRESS),검색어 입력 후 엔터키 입력 시 조회 동작 확인,검색어 입력 + Enter 키,검색 버튼 클릭과 동일하게 조회가 수행된다.
유효성검사 (VALIDATION),날짜 범위 7일 초과 시 검증 확인,"시작일: 2026-01-01, 종료일: 2026-01-15","경고 메시지 ""조회 기간은 최대 7일입니다"" 표시 및 조회 차단"
유효성검사 (VALIDATION),필수 선택 항목 미선택 시 검증 확인,필터 선택 없이 조회 버튼 클릭,오류 메시지 표시 및 조회 차단
필터변경 (ONCHANGE),필터 선택 시 자동 조회 확인,상태 필터 '완료' 선택,선택한 필터 조건으로 즉시 조회되고 그리드가 갱신된다.
페이지사이즈 (ONCHANGE),페이지 당 건수 변경 시 확인,20건/50건 선택,페이지 사이즈가 변경되고 페이지가 1로 초기화된다.
페이지네이션 (ONCHANGE),페이지 번호 클릭 시 목록 갱신 확인,2페이지 클릭,해당 페이지의 데이터로 그리드가 갱신된다.
정렬 (ONCLICK),컬럼 헤더 클릭 시 정렬 동작 확인,날짜 컬럼 헤더 클릭,날짜 기준 오름차순/내림차순 정렬이 적용된다.
엑셀다운로드 (ONCLICK),엑셀 다운로드 버튼 클릭 시 동작 확인,다운로드 버튼 클릭,현재 조회 조건의 전체 데이터가 엑셀 파일로 다운로드된다.
에러처리 (ONERROR),API 호출 실패 시 처리 확인,네트워크 오류 발생,사용자에게 오류 메시지가 표시되고 로딩 상태가 해제된다.
로딩상태 (ONLOADING),데이터 로딩 중 UI 처리 확인,API 응답 대기 중,로딩 스피너 또는 스켈레톤 UI가 표시된다.
엣지케이스 (EDGE),조회 결과 0건인 경우 확인,존재하지 않는 조건으로 검색,"""조회된 데이터가 없습니다"" 메시지가 그리드에 표시된다."
초기화버튼 (ONCLICK),초기화 버튼 클릭 시 검색 조건 리셋 확인,초기화 버튼 클릭,모든 검색 조건이 기본값으로 초기화되고 그리드는 유지된다.
```

## CSV 특수문자 처리

CSV 형식에서 쌍따옴표(")가 포함된 경우:

**규칙:** 쌍따옴표를 두 번 연속(`""`)으로 작성하여 이스케이프

**예시:**
```csv
유효성검사 (VALIDATION),날짜 범위 초과 시 검증,"8일 범위 선택","경고 메시지 ""조회 기간은 최대 7일입니다"" 표시"
엣지케이스 (EDGE),빈 결과 처리,존재하지 않는 ID 검색,"""데이터가 없습니다"" 메시지 표시"
```

## 작성 체크리스트

CSV 작성 완료 후 아래 항목을 점검:

- [ ] 모든 행이 4개 컬럼을 포함하는가?
- [ ] 분류에 이벤트 타입이 포함되어 있는가?
- [ ] 점검 항목이 "~~ 확인" 형태인가?
- [ ] 테스트 데이터가 구체적이고 실행 가능한가?
- [ ] 기대 결과가 관찰 가능한 UI/상태 변화인가?
- [ ] 쌍따옴표가 올바르게 이스케이프되었는가?
- [ ] 다중 파일 분석 시 페이지별로 구분되어 있는가?
